description: Updates CHANGELOG.md following Keep a Changelog format.
---

Update CHANGELOG.md with entries for recent work not yet documented:

1. Read the current CHANGELOG.md (or create it if missing).
2. Run git log --oneline since the last documented commit.
3. Group commits by type (Added, Changed, Fixed, Removed) using human-readable entries.
4. Place new entries under ## [Unreleased] and follow formatting guidelines.
